# Security Policy

## Supported Versions

We release security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Do not open public issues or pull requests for security vulnerabilities.**

Instead, please report security issues by emailing **security@lexora.app** with:

1. Description of the vulnerability
2. Steps to reproduce (if applicable)
3. Potential impact
4. Your suggested fix (if you have one)

### What to expect:

- Acknowledgment within 48 hours
- Initial assessment within 1 week
- Coordinated disclosure timeline (typically 90 days)
- Credit in security advisory (if desired)

## Security Best Practices

We recommend users:

- Keep LEXORA and its dependencies **updated**
- Use **HTTPS** when deploying to production
- Set strong **environment variables** for sensitive data
- Enable **GitHub security scanning** if self-hosting
- Report suspicious activity immediately

## Dependency Scanning

We use automated tools to monitor dependencies for known vulnerabilities:

- GitHub Dependabot
- npm audit

Regular security updates are published as patch releases.
